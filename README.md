# README

## Описание проекта
Данный проект содержит несколько функций для работы с данными, такими как транзакции, номера карт и счета. 
Функции позволяют фильтровать, сортировать данные и маскировать чувствительную информацию.

## Установка
Проект не требует дополнительной установки, так как написан на Python и использует стандартные функции языка.

## Описание функций

### `main()`
Функция запускает другие функции и выводит результат их работы.
**Параметры:**
- нет
- **Возвращает:**
- Списки транзакций в виде списка словарей.

### `save_report_to_file(filename: Optional[str] = None):`
Декоратор для сохранения результатов функции в файл.
Если имя файла не указано, генерируется автоматически в формате: 'spending_report_YYYY-MM-DD.txt'
**Параметры:**
- `filename` (Optional[str] = None)

### `spending_by_category(df_data: pd.DataFrame, chosen_category: str, end_date: Optional[str] = None)`
Считает сумму операций по указанной категории за последние 3 месяца.
**Параметры:**
- `df_data` (pd.DataFrame): DataFrame с колонками ["Категория", "Сумма операции", "Дата платежа", "Статус", 
"Валюта операции"]
- `chosen_category` (str): Категория для фильтрации
- `end_date` (Optional[str]): Конечная дата в формате строки (если None - используется текущая дата)
- **Возвращает:**
- Словарь, в котором сумма в рублях операций по категории за период

### `get_search_by_mobile_numbers(transactions_data: dict)`
Возвращает транзакции, у которых есть в описании номер телефона, в виде JSON.
**Параметры:**
- `transactions_data` (dict) — JSON файл с транзакциями
- **Возвращает:**
- Новый список словарей, содержащий отфильтрованные транзакции.

### `load_user_settings(file_path: str)`
Загружает настройки из JSON файла.
**Параметры:**
- `file_path` (str) — путь к файлу JSON с настройками.
- **Возвращает:**
- Новый список словарей, содержащий настройками.

### `get_currency_rate(currency_code: str, payment_date: str)`
Получает курс валюты к рублю на указанную дату.

**Параметры:**
- `currency_code` (str) — Код валюты из 3 букв ('USD', 'EUR' и т.д.).
- `payment_date` (str) — Дата в формате 'ДД.ММ.ГГГГ' (например '24.05.2023').

**Возвращает:**
- (float) — Курс валюты к RUB или 0.0 при ошибке

### `read_excel_file_to_df(file_path: str)`
Читает данные из Excel-файла и возвращает дата фрейм.

**Параметры:**
- `file_path` (str) — Путь к Excel-файлу.

**Возвращает:**
- (pd.DataFrame) — дата фрейм с данными (в случае успеха) или сообщение об ошибке (при возникновении исключения).

### `get_greeting()`
Получает от системы местное время, возвращает приветствие в зависимости от времени.

**Возвращает:**
- (str) — Возвращает приветствие в зависимости от времени.

### `get_card_statistics(transactions_data: dict)`
Обрабатывает JSON. По каждой карте выводит последние 4 цифры карты, общую сумму расходов, кэшбэк.

**Параметры:**
- `transactions_data` (dict) — JSON словарь.

**Возвращает:**
- (List[Dict[str, Union[str, float]]]) — JSON словарь.

### `get_top_transactions(transactions_data: dict)`
Обрабатывает JSON. Выводит топ-5 транзакций по сумме платежа.

**Параметры:**
- `transactions_data` (dict) — JSON словарь.

**Возвращает:**
- (List[Dict[str, Union[str, float]]]) — JSON словарь.

### `get_currency_rates()`
Выводит курс валют. Список валют хранится во внешнем JSON словаре.

**Возвращает:**
- (List[Dict[str, Union[str, float]]]) — JSON словарь.

### `get_stock_prices`
Выводит курсы акций. Список компаний хранится во внешнем JSON словаре.

**Возвращает:**
- (List[Dict[str, Union[str, float]]]) — JSON словарь.

### `generate_finance_report(data: dict)`
Генерирует финансовый отчёт в формате JSON.

**Параметры:**
- `data` (dict) — список словарей с транзакциями.

**Возвращает:**
- (str) — Возвращает JSON.

## Тестирование

Проект покрыт тестами более чем на 80%. Для тестирования используется `pytest` и `pytest-cov`.

- `test_main`: Проверка корректности главной функции.
- `test_views`: Проверка views.py.
- `test_services`: Проверка services.py.
- `test_utils`: Проверка utils.py
- `test_reports`: Проверка reports.py

### Запуск тестов

```sh
pytest
```

### Генерация отчета о покрытии тестами

```sh
pytest --cov=your_module --cov-report=html
```

После выполнения команды откройте `htmlcov/index.html` в браузере для просмотра детального отчета.
